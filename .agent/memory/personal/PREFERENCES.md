# Personal Preferences

> **This file is yours.** It's the one file every new user should customize.
> Preferences are context, not procedure — tell the agent who you are, not
> how to write code.

## Code style
- Swift / SwiftUI for UI, SceneKit for 3D visualization
- Prefer small, focused changes over large refactors
- Memory usage is the top priority — always profile before assuming
- Direct state mutations over unnecessary abstraction

## Workflow
- Verify build passes (`swift build`) after every change
- Keep PRs small and independent
- Raise concerns immediately rather than building toward a wrong solution

## Constraints
- Primary stack: Swift, SwiftUI, SceneKit, NSCollectionView
- Comic formats: CBZ, CBR, PDF
- Performance target: smooth 60fps scrolling, minimal RAM footprint
- No force push to main or production branches

## Communication
- Be direct, no filler, no pleasantries
- Surface tradeoffs explicitly — don't hide them
- Challenge weak reasoning immediately
- No emojis
